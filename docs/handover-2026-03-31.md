## Important Discovery

- Direct `curl` scraping of Streamline set pages is blocked by a Vercel security checkpoint.
- A real browser context works.
- The public set pages expose useful structured data via `window.__NEXT_DATA__.props.pageProps.initialState.streamlineApi.queries`.
- That embedded data is good enough to extract:
  - family metadata
  - categories
  - grouped icon records
  - per-icon SVG
  - tags
  - source download page URL

## Current Blocker

- The set page only includes the first lazy-loaded chunk of icons in the embedded query state.
- Example for `core-line-free`:
  - initial embedded payload only yielded `50` icons
  - pagination reported:
    - `total: 998`
    - `offset: 0`
    - `nextSkip: 50`
    - `hasMore: true`
- Because of that, a first real `download-set core-line-free` run produced only a partial pack.
- That partial pack was deliberately removed again and is **not committed**.
